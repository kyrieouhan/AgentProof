# M3：浏览器验证与防作弊轻量版，本地 MVP

## 目标

在 M2 确定性 API/证据闭环上补齐一条关键用户流程、页面与数据一致性、测试/配置 Diff 和轻量硬编码检测，形成可在本地使用的 MVP。

## 前置依赖

M2 全部门禁通过；API 与 Manifest 稳定；关键用户流程已有经用户确认的验收标准；M1 隔离回归继续通过。

## 负责人

全栈/测试开发主责；Runner 负责人维护浏览器和目标服务隔离；安全评审负责页面、下载、网络和 Trace 数据风险。

## 输入

M2 运行、已确认浏览器步骤、稳定定位信息、角色/测试数据、官方 Demo 正确与缺陷版本、基线测试/CI/Mock/Fixture Diff。

## 工作内容

- 以结构化步骤执行一条关键 Playwright 用户流程；优先可访问名称和稳定定位器，并允许用户编辑。
- 采集关键/失败截图、控制台错误、失败网络请求和可选 Trace。
- 联合 API 与数据库证据验证页面、接口和持久化状态一致。
- 比较原有测试、CI、Mock、Fixture 和断言变化，只输出风险和具体 Diff，不直接指控作弊。
- 只读挂载独立验收目录；随机化数据与执行顺序以检测固定账号/ID 硬编码。
- 对同一 Commit 重复运行 3 次；结论不一致使用 `unstable` 并保留种子与 Trace。

## 交付物

Playwright 验证器、关键流程 Schema/编辑机制、只读验收目录、测试与配置 Diff、轻量硬编码检测、完整验收报告、可分享 HTML/Markdown 导出和回归套件。

## 明确非目标

不做通用 Mutation Testing、自动修复、任意框架兼容、GitHub App、真实用户 Alpha、云端 Runner、企业多租户或完整多流程浏览器测试平台。

## 量化退出门禁

- 官方缺陷版本中的表面完成、权限漏洞、测试削弱、硬编码、状态不持久、API 失败但 UI 成功均能稳定发现，且正确版本不被误判。
- 同一 Commit 重复 3 次，核心结论一致；否则使用 `unstable` 并保存种子、环境和 Trace。
- 报告逐项展示状态和原始证据；阻断级安全失败覆盖总体完成度。
- 一名未参与实现的内部评审者能按文档运行官方 Demo 并定位失败证据；这不等于外部用户或市场验证。
- M1 隔离烟测与 M2 脱敏/Manifest 回归全部通过。

## 安全要求

浏览器上下文隔离；禁用任意下载、外部导航和危险协议；限制 Trace/截图；敏感页面脱敏；目标仓库不能写验收目录或最终证据；网络策略不因浏览器加入而默认放宽。

## 风险

定位器脆弱、页面延迟、Trace 泄密、正常测试修改被误判、随机化改变语义、浏览器零日。无法可靠验证时使用 `unverifiable`，重复不一致用 `unstable`。

## 推荐任务拆分

1. 浏览器步骤 Schema 与单流程执行。（已完成 M3-01）
2. 截图/网络/控制台/Trace 采集和脱敏。（已完成 M3-02；Trace 默认禁用、显式 opt-in）
3. 页面/API/数据库联合断言。（已完成 M3-03）
4. 测试与配置 Diff 风险模型。（已完成 M3-04）
5. 只读规则与随机化硬编码检测。（已完成 M3-05）
6. 官方 Demo 缺陷回归、3 次一致性与内部可用性检查。（已完成 M3-06）
7. 最小本地 Web 用户入口。（已完成 M3-07）

## 当前实现进度

- M3-01 已完成：新增 `BrowserFlowSchema`、`browser-flow.schema.json`、单流程执行器和 `npm run m3:browser-smoke`。
- 官方 Demo smoke 使用本机 Chrome，经 `playwright-core` 执行注册、登录态确认、任务写入和普通用户管理员接口拒绝检查。
- 本阶段顺手修复了 Demo 正确基线中任务表单异步提交后错误读取 `event.currentTarget` 的问题；该问题由真实浏览器 smoke 暴露。
- M3-02 已完成：browser smoke 采集 console/page error/failed request/failed response 事件、脱敏最终截图、evidence refs 和 Manifest。
- Trace 支持通过 `AGENTPROOF_BROWSER_TRACE=1` 显式启用；默认禁用，避免在未完成更强 Trace 脱敏策略前保存 DOM/输入快照。
- M3-03 已完成：官方 Demo smoke 对同一任务同时验证页面可见、同会话 API 返回和 SQLite 持久化记录，并把 `joined-observation.json` 纳入 evidence refs。
- M3-04 已完成：新增 Diff 风险模型与 `npm run m3:diff-risk`，对官方 `weakened_tests` 补丁输出 high/human_review 风险和具体测试位置；该模型只提示风险，不直接指控作弊。
- M3-05 已完成：新增只读规则检查和 `npm run m3:hardcoded`，对官方 `hardcoded_behavior` 补丁输出随机等价输入失败、固定 demo 输入成功的 human_review 风险。
- M3-06 已完成：新增 `npm run m3:regression` 和 `artifacts/m3-regression/`，浏览器正确基线 3 次一致，五类官方缺陷 5/5 复现。该阶段是 CLI/回归/证据收口，不等同于可操作 Web 界面。
- M3-07 已完成：新增 `src/web/`、`npm run web:dev`、`npm run web:start` 和 `npm run m3:web-smoke`；用户可从本地 Web 页面导入官方 Demo、编辑验收项、启动真实 Runner/M3 验收、查看阶段进度、打开截图/日志/API/数据库/Manifest 证据，并导出 HTML/Markdown 报告。Web smoke 连续 3 次通过且结论一致。

## 里程碑边界

- M2 提供 API、数据库、Manifest 与状态模型；M3 只增加本地浏览器与轻量防作弊。
- M4 才邀请真实用户、运行外部项目、发布 Alpha、接入 GitHub App 或建设完整多项目 Web 平台。
