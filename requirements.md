# AgentProof 需求基线

## 背景与目标

AgentProof 为 AI 编程 Agent 的交付提供独立、可复现、证据驱动的验收层。本阶段只建立长期可维护的项目文档、里程碑、威胁模型、支持矩阵和 failure-case 数据规范。

## 本次必须完成

- 将原始 Word 计划准确转换为 Markdown，并保留架构与阶段信息。
- 固化 M0-M4 顺序、MVP 边界、统一状态与安全基线。
- 把 M0 改为问题与技术可行性验证，不要求采访或付费验证。
- 建立威胁模型、支持矩阵、ADR 和 failure-case Schema。

## 本次明确不做

不开发 Runner、API、前端、模型接入、需求解析器、Playwright 验证器、GitHub App、Monorepo 或任何占位业务代码；不安装项目依赖；不虚构反馈、案例来源或测试结果。

## 验收标准

- [x] 原始 DOCX 完整读取并保留。
- [x] 创建总计划、M0-M4、威胁模型、支持矩阵、ADR、术语表和数据集骨架。
- [x] 统一底层状态与合并建议。
- [x] 文档无前期用户采访门禁。
